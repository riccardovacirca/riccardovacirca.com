Introduzione

APRX è una libreria di estensione di APR (Apache Portable Runtime) e ha lo scopo di aggiungere a quest'ultimo, funzionalità in esso non presenti e promuoverne l'adozione in ambito web. Questa pagina documenta le funzioni di APRX e ne mostra esempi d'uso in un contesto strutturato. Lo scenario di riferimento è quello di un modulo di estensione del server HTTP Apache.

Organizzazione di un progetto

Un progetto APRX è organizzato secondo due sole entità di programma: funzioni base e funzioni della logica applicativa. Le prime sono funzioni di utilità e funzioni helper. Le seconde implementano la logica applicativa vera e propria. In questo contesto una applicazione è concepita essenzialmente come un insieme di moduli di estensione del server http Apache2. In questo contesto sono implementate 4 entità: operazione, funzione, servizio, applicazione

Funzioni base

Le funzioni base implementano le operazioni atomiche e isolabili della logica applicativa, come ad esempio la lettura e scrittura da un database, e comprendono anche le funzioni di utilità. Il codice della funzione contiene solo la logica essenziale per l'operazione da implementare. Di seguito è mostrata ad esempio una funzione che esegue il join di un array in usa stringa. 

// Base function example
char* join(apr_pool_t *mp, apr_array_header_t *ar) {
  char *result = NULL, *s;
  if (ar->nelts > 0) {
    s = APR_ARRAY_IDX(ar, 0, char*);
    if (s) {
      result = apr_pstrdup(mp, s);
    }
  }
  if (ar->nelts > 1) {
    for (int i = 1; i < ar->nelts; i ++) {
      s = APR_ARRAY_IDX(ar, i, char*);
      if (s) {
        result = apr_pstrcat(mp, ",", s, NULL);
      }
    }
  }
  return result;
}

Funzioni della logica applicativa

Le funzioni della logica applicativa contengono tutta la complessità relativa ad una operazione di livello applicativo. In questo ambito ricadono ad esempio le operazioni di registrazione di un nuovo utente, una richiesta di autenticazione, ecc. Queste funzioni utilizzano funzioni base per implementare le operazioni richieste e implementano il livello applicativo. 

int main(int argv, char **argv) {

  // Inizializzazione del valore di ritorno
  int result = 0;

  // Inizializzazione dello stato
  struct {
    int input, init, pool;
  } st = {0, 0, 0};

  // Validazione dell'input
  st.input = (argv >= 2) && (strcmp(argv[1], "test") == 0);

  if (st.input) {
    st.init = apr_initialize() == APR_SUCCESS;
  }
  
  apr_pool_t *mp;
  if (st.init) {
    st.pool = apr_pool_create(&mp, NULL) == APR_SUCCESS;
  }

  if (st.pool) {
    result = 1;
  }

  st.error = result != 1

  if (st.error) {
    if (!st.init) {
      printf("Errore di inizializzazione.\n");
    } else if (!st.pool) {
      printf("Errore di allocazione del pool di memoria.\n");
    } else {
      printf("Errore generico.\n");
    }
  }

  if (st.pool) {
    apr_pool_destroy(mp);
  }

  if (st.init) {
    apr_terminate();
  }

  exit(EXIT_SUCCESS);
}
